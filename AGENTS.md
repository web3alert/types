# Types Repository - Контекст задачи

## Подзадача репозитория

Поддерживать типы данных для v2 сущностей проектов и триггеров, используемых UI/API в проектном сценарии подписок.

## Текущие наработки

- В рамках текущей задачи изменения в `types` не вносились.
- Типы `Project`/`ProjectView` и `Trigger` используются как контракт между UI и API.
- Текущий flow UI (проекты + триггеры проекта) опирается на существующие модели без доработок.

## Примечание

При расширении логики prefill подписки по trigger может потребоваться уточнение типизации query/options в UI-контрактах.
